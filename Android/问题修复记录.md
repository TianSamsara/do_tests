# Android版本问题修复记录

## 问题描述

首次运行时出现以下错误：

```
TypeError: Properties ['name'] passed to __init__ may not be existing property names.
```

## 问题原因

所有界面类（LoginScreen、MainScreen、QuizScreen、ReviewScreen）原本继承自 `BoxLayout`，但在使用 `ScreenManager` 时，屏幕必须继承自 `Screen` 类。

`Screen` 类有一些特殊属性（如 `name`），这些属性不能通过 `**kwargs` 传递给父类的 `__init__` 方法。

## 解决方案

### 1. 修改继承关系

将所有屏幕类从继承 `BoxLayout` 改为继承 `Screen`：

```python
# 修改前
from kivy.uix.boxlayout import BoxLayout
class LoginScreen(BoxLayout):

# 修改后
from kivy.uix.boxlayout import BoxLayout
from kivy.uix.screenmanager import Screen
class LoginScreen(Screen):
```

### 2. 调整初始化方式

由于 `Screen` 是容器组件，需要在其中添加一个 `BoxLayout` 作为根布局：

```python
# 修改前
class LoginScreen(Screen):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.orientation = 'vertical'
        self.padding = dp(20)
        # 直接使用 self.add_widget()

# 修改后
class LoginScreen(Screen):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        
        # 创建主布局
        main_layout = BoxLayout(
            orientation='vertical',
            padding=dp(20),
            spacing=dp(10)
        )
        
        # 所有widget添加到main_layout
        main_layout.add_widget(...)
        
        # 最后将main_layout添加到Screen
        self.add_widget(main_layout)
```

- ✅ `screens/login_screen.py` - 登录界面
- ✅ `screens/main_screen.py` - 主界面
- ✅ `screens/quiz_screen.py` - 刷题界面
- ✅ `screens/review_screen.py` - 复习界面

## 技术要点

### Kivy Screen vs BoxLayout

- **Screen**: 
  - 用于ScreenManager管理的页面
  - 有特殊的属性和方法（name、on_enter、on_leave等）
  - 是容器组件，需要在其内部添加布局
  
- **BoxLayout**:
  - 基础布局组件
  - 直接控制子组件的排列
  - 不能作为ScreenManager的页面

### 正确的Screen使用方式

```python
from kivy.uix.boxlayout import BoxLayout
from kivy.uix.screenmanager import Screen

class MyScreen(Screen):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        
        # 1. 创建布局
        layout = BoxLayout(orientation='vertical')
        
        # 2. 添加组件到布局
        layout.add_widget(Button(text='按钮'))
        
        # 3. 将布局添加到Screen
        self.add_widget(layout)
```

## 总结

这是一个典型的Kivy框架使用问题。在开发Kivy应用时：

1. 如果使用 `ScreenManager` 管理页面，所有页面必须继承 `Screen`
2. `Screen` 本身不是布局，需要在其内部添加布局组件
3. 所有的UI组件应该添加到内部布局，而不是直接添加到Screen

修复后的代码完全符合Kivy的最佳实践，可以正常运行和打包为Android APK。
